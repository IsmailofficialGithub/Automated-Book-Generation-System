Trial Task - Full Stack AI Developer
Automated Book Generation System
Objective:
Your task is to begin building a modular and scalable book-generation system. The system should accept a title, generate an outline, write chapters using feedback-based gating logic, and compile the final draft. The design must support human-in-the-loop notes for outline and chapters and conditional branching at every stage.

You can use any combination of input file, DB, python scripts or node based tool, any LLM for this task. Please mention your complete stack when submitting.

The goal is to show how you'd approach architecting this automation, build the first parts, and optionally suggest better approaches. We assess clarity of logic, modular design, proper use of context, and Supabase integration.

Workflow Stages
1. Input + Outline Stage

This stage requires you to take input from a file and create an outline from an LLM and store it to DB for review by the editor. The editor can add notes next to the title to generate an outline. Your system should take both into account. The editor can add notes to generated outline which can be used to regenerate the outline to make it even better.


Input Field

Notes

title

Mandatory

notes_on_outline_before

Required before generating outline

outline

Generated by GPT, based on title + notes

notes_on_outline_after

Optional post-outline input

status_outline_notes

yes, no, no_notes_needed

Logic:

Only generate outlines if notes_on_outline_before exists.
After outline: check status_outline_notes
yes: wait for notes.
no_notes_needed: proceed.
no/empty: pause.
2. Chapter Generation Stage

From the outline, you must now generate chapters. Each chapter is generated separately and needs to be reviewed by the editor. Each new chapter must have context of previous chapters to understand the whole book. You need to create a logic for past chapter context to be used for the next chapter generation. Like the outline, editor can add notes to regenerate the chapters to make them better.

Input Field

Notes

outline

Used to generate chapters

chapter_notes_status

yes, no, no_notes_needed

chapter_notes (optional, per chapter)

Specific chapter notes

chapter_summaries

Stored per chapter for context chaining

Logic:

For each chapter:
Use summary of all previous chapters as context.
If chapter_notes_status = yes, wait for notes.
If no_notes_needed, proceed.
If no/empty`, pause.
3. Final Compilation Stage

Once all chapters have been generated and reviewed, you must compile them into the final draft of the book in word, pdf, or txt format.

Input Field

Notes

final_review_notes_status

yes, no, no_notes_needed

book_output_status

Tracks if book is ready or paused

Logic:

Compile only if:
final_review_notes_status = no_notes_needed
OR notes exist for final draft
Export as .docx or .txt
Notification System
Trigger email or MS Teams webhook on key events:

Outline is ready and needs review
Waiting for chapter notes
Final draft is compiled
Error or pause due to missing input
Chapter Context Handling (Summaries)
Before generating chapter N, summarize chapters 1 to N–1 and pass that as context_input to GPT.

Example prompt:

"Using the following chapter summaries, write Chapter 4 of the book titled X..."

Store summaries in Supabase to re-use or regenerate if needed.
